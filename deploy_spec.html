必须 · deploy/Dockerfile
标准 Dockerfile，用于构建项目镜像。平台默认以此文件构建，无需其他脚本。
若仓库根目录存在 Dockerfile，也会被识别（优先级低于 deploy/Dockerfile）。 
# 通用示例
FROM python:3.11-slim
WORKDIR /app
COPY . .
RUN pip install -r requirements.txt
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "80"]
工程类型 · Python（自动 base stage 缓存）
工程类型设为 Python 时，平台在每次构建前自动检测 requirements.txt 是否变化： 未变化则跳过 pip install，直接复用已有 base 镜像；变化则先重建 base stage，再构建 app。

需将 Dockerfile 改为 multi-stage，用命名 stage 拆分依赖层与业务层： 
# Stage 1: base（依赖层，requirements.txt 不变时复用缓存）
FROM python:3.9-slim AS base
RUN apt-get install -y ... && pip install --upgrade pip
COPY requirements.txt .
RUN pip install -r requirements.txt

# Stage 2: app（业务层，每次部署重新 COPY 代码）
FROM base AS app
WORKDIR /app
COPY . .
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "80"]
base 镜像名为 {服务名}-base:latest，hash 通过 docker image label req_hash 追踪。 无需编写 pre_build.sh，平台自动处理。
工程类型 · Java（Maven / Gradle）
工程类型设为 Java 时，平台自动在临时容器内执行 Maven 或 Gradle 构建：
• 检测到 pom.xml → 使用 maven:{java_version}-eclipse-temurin 镜像执行 mvn package -DskipTests
• 检测到 build.gradle / build.gradle.kts → 使用 gradle:{java_version}-jdk 镜像执行 gradle build -x test
• 支持 gradlew wrapper（自动赋予执行权限）
• 可通过「自定义构建命令」覆盖默认命令

构建完成后，平台使用 Dockerfile 打包最终镜像（JAR 已在 target/ 或 build/libs/ 中）： 
# Java 项目 Dockerfile 示例
FROM eclipse-temurin:17-jre-alpine
WORKDIR /app
COPY target/*.jar app.jar
EXPOSE 8080
CMD ["java", "-jar", "app.jar"]

# Gradle 项目
FROM eclipse-temurin:17-jre-alpine
WORKDIR /app
COPY build/libs/*.jar app.jar
EXPOSE 8080
CMD ["java", "-jar", "app.jar"]
Java 版本默认 17，可在工程配置中修改。
工程类型 · 前端（npm → nginx）
工程类型设为 前端 时，平台在临时 node 容器内执行构建， 然后将 dist 目录的产物通过 docker cp 推入 nginx 容器或直接复制到宿主机目录。
无需 Dockerfile，只需配置构建命令和目标路径。
可选 · deploy/hooks/ 钩子系统
在部署流程的关键节点注入自定义逻辑，脚本不存在则静默跳过。 脚本以 bash 执行，非零退出码中止部署。
pre_build.sh docker build 前执行。适合：自定义构建准备、生成配置文件等。
post_build.sh docker build 后执行。适合：冒烟测试、推送镜像到 registry。
pre_deploy.sh compose up 前执行。适合：数据库迁移、数据备份。
post_deploy.sh 健康检查通过后执行（仅非自重启场景）。适合：缓存预热、通知。
钩子环境变量
平台自动注入以下变量到每个钩子脚本。
IMAGE目标镜像名（如 myapp:latest）
ROLLBACK_TAG回滚镜像名（如 myapp:rollback）
SERVICEdocker-compose 服务名
COMPOSE_FILEdocker-compose.yml 路径
REPO_DIR临时 clone 目录（含完整代码）
PROJECT_ID平台工程 ID
部署流程
git clone --depth=1 $REPO_URL /tmp/deploy_xxx

# 备份当前镜像为 :rollback
docker tag $IMAGE $ROLLBACK_TAG

# 构建阶段
bash deploy/hooks/pre_build.sh    # 可选

# Python: 自动检测 requirements.txt 变化 → 按需重建 base stage
# Java:   在临时 Maven/Gradle 容器内构建 → 产物复制回宿主机
# 前端:   在临时 node 容器内 npm build → 产物复制回宿主机

docker build -t $IMAGE -f deploy/Dockerfile .  # Python/Java/通用
bash deploy/hooks/post_build.sh   # 可选
